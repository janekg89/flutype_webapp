How to prepare content for database ?

Overall View

Study

Here, I explain how the content has to be prepared to fit into the database. Therefore, I start with the biggest unit in the database, the study . A study contains a collection of measurements and it has a task. The measurements are conducted to investigate that task. In figure to the left the schematic structure of a study is shown. I go through the figure in detail. This structure obviously corresponds to the structure of the data for the import. Therefore, I recommend to read this first before examining the different examples of how to import data.

By conducting a study an unique identification string (sid) has to be defined. The recommendation is to start the string with a date. Additionally a task of the study should be declared and written into the comment. Any related files can be attached to the study. These files can be thought of a the raw documents. Most importantly, any number of measurements can be linked to a study. For each measurement thereby, an unique identification string (sid), an batch sid , measurement type , fixed ligand distribution, analyt distribution , comment , process and the raw docs can be defined. In the following, I will go through each point to clarify what I mean.

  • sid (mandatory)- is an unique identification string.
  • batch sid - is an unique identification string for a batch of microarrays or microwell plates.
  • measurement type - can be chosen from microarray, microwellplate and elisa.
  • fixed ligand distribution
    • (mandatory)- are described by the gal file of the ligands which are immobilized / spotted. Alternatively, a table with rows and columns can be given. The content of the fields shell be the sids of the ligand batches. This is described in ligands section.
  • analyt distribution (mandatory)- has the same format as the fixed ligand distribution. Howerver, the distribution of the analyt. Most often the gal file will be filled with the virus batch sid
  • comment - is given any additional information to the experiment.
  • process (mandatory)- an excel sheet with ordered steps in the process. How to define steps in an process is described in the process section
    • and is probably better to understand by checking the examples.
  • raw docs - are any files corresponding to the measurement additionally to the existing information.

Ligands

Ligands can be either antibodies, peptides, viruses or a complex which is a set of any of the ligands. The biggest unit in the ligands scheme is the ligand batch. It links to one of described ligands but also adds informations about the labeling, concentration, buffer, ph, purity, production date, pruduced_by and a comment.

Process

The process containes an ordered list of steps.The possible steps are of the type: scanning, quenching, spotting, blocking ,drying , incubating and washing. the same steps can be added multiple times into the process. Think for example of the same washing procedure which is performed multiple times during the process. Further, a starting time, finishing time, image, intensity file and an comment can be linked to a specific step in the process. Image and intensity linking is espacially meant to for the scanning step, which should provides either an image or intensity file.
