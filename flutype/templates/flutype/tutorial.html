How to Upload Data?

The concept of uploading data into the database can be devided into two parts. On the one hand, one has to fill in general data, which is important for all measurements and not related to a specific measurement. Ligands, ligands batches and process steps are these kind of data. It is possible to add/modify this content directly in the browser. On the other hand, there are measurement specific information, which use the general data. The measurement specific information has to be saved with a specific pattern into a directory. This directory can finally be used to upload the measurements. While creating such a directory, it is important to check if all ligands, ligand batches and process steps are already in the database. This tutorial shows the workflow. The biggest unit in the database is a study. It refers to a collection of measurements, which were measured collectively for a reason. In this tutorial, I will show an example of how to upload a study. This study aims to show how to prepare the data to upload it into the database. Hence, it is reasonable to upload a study containing three unequal measurements. The directory with the study is ziped and can be downloaded here . The related directory tree can be seen here. Now, I will give you a step by step tutorial. Let's start.

1. Study

A study contains meta data, measurements and raw docs. Let's start with the meta data. Meta data contains an identification string (sid) for the study and a comment. For the sid I recommend to find a convention for the naming of the studies to not get name conflicts. Since our study is an tutorial, I name my study: "170929-tutorial". The task of a study is described in the results field and the results of the study are described in the results field. Don't worry if you not have any results yet. They can be updated afterwards. Results related documents can be uploaded into the folder raw docs. The raw docs folder will be explained later. Now back to the meta data of our exemple. The following table shows the meta data of my study. this table is saved as meta.tsv in the main directory of the study.

sid 170929-tutorial
task The task of the study is to demonstrate how a study can be uploaded into the database. Therefore, three different types of experiments are included into the study. The aim is to select very different kinds of experiments to get an preferably complete demonstration of what can be uploaded and how.
result
Allright, we have defined a study. Most importantly we can add measuremnts to our study. This is going to be covered in the next section. But further, it is possible to upload any additional documents related to the study. To do so, just paste them into the "raw docs" folder of the study. In this study, I add the zipped folder of the study to the raw docs. Now, let's go ahead and add some measurements to the study.

2. Elisa Measurement

For any measurement the same five informations have to be given: meta data, ligands, process, results and raw docs. We start with an elisa measurement without any peptides involved. We start of with the most general information, the meta data.

2.1 Elisa Meta

sid 170929-tutorial-elisa-1
batch_sid 12345689

Alright, let's go quickly through the table. The sid is a unique id of the measurement. Again, make sure there is no name conflict with existing sids. Any sid introduced in the following has to be unique! The batch sid refers to the batch of the microwell plate. The measurement type has to be chosen from three choises. elisa, microarray and microwell. Both, microarray and microwell refer to fluoresence measurements. The user can be chosen from the users list . The surface substance refers to the functionalization of the microwell plate. In this measurement no ligands were bound to the surface of the microwell plate. Therefore, also no fuctionalization was necessary. We enter a "No", which is different to an empty space. An empty space refers to an unknown value ( NaN ). The manufacturer refers to the manufacturer of the microwell plate. These are the meta data of a measurement. This table is saved as "meta.tsv" in the main directory of a measurement folder. Let's continue with the next important measurement related information, the ligands and their positions on the microwell plate.

2.2 Elisa Ligands

Let's start with some general information related to ligands. Peptides, antibodies, viruses and any combination of several ligands (complexes) are referred to as ligands. Each of them is an ligand type and has its specific information. A peptide, for example, is described by a linker, spacer, sequence and c-terminus. Besides the specific information, all ligands share the fields: sid and comment. However, it might be interesting also to conduct measurements with e.g. different concentrations of the ligands, which is not captured by the ligands related informations. Here, the concept of ligand batches comes into play. A ligand batch links a ligand to the following batch related properties: labeling, concentration, buffer, pH, purity and production date. Also for ligand batches, a sid has to be defined. Keep in mind that ligand batches are particularly important, because their sids are later on used for referencing in the gal files. Ligand batches, can be of five different types depending on what kind of ligand they are linked to: virus batch, peptide batch, antibody batch, complex batch and buffer batch. The buffer batch is a batch without a link to a ligand. The remaining four should be self explaining.

Now back to our measurement. In this example we do not immobilize any ligands on the surface. This reduces our work. No, need to define any peptides or peptide batches. However, still we are supposed to deliver a so called gal file, which maps a ligand batch to a location on a microwell plate. Since no ligands are present in any well, we fill all entries in the table with "No". Note that the table has to have the same dimensions as the microwell plate. For microarrays it might be more reasonable to provide a gal file, which is fine too. This table is saved as "lig_fix_XXX.txt" in the main directory of the measurement.

A process step can be of various types: blocking, drying, incubating-analyt, incubating, quenching, scanning, spotting and washing. Each step is described by a sid , method, temperature and comment. Some of the steps take also the information substance and duration. Let's take a look at each step.

Any defined process step is reusable also in different processes. Thus, it is a general information. A process can contain many steps in an arbitrary ordering and with additional experiment specific informations. We explain this concept based on our example. So, let's go back to our elisa example.

2.4. Elisa Results

In this section, we are going to deal with informations related to the measurement results. Measurement results describe the intensities resulting from the measurement. In this example we measured the intensities during the last step of our process and saved it as "intensity_final.tsv" in the main folder of the measurement. If we want to use the file we have to save it again in a folder within the results folder. It might seam to be a redundant procedure. However, the aim is to decouple the results from the process. In the microwell example, this approach might become clearer. There, multiple scans have been performed during one process. From theses scans three different results have been optained. Often in might be interessting to subtract the background or to do even more sophisticated post processing of the results. This concept makes it possible to save the results in the database aswell. of the measured The measurement results meta and intensity.

3.4 Microwell Results

In this example we want to show how to upload three results. The first result will be the intensities from the the first scanning (process step 7), the second result will be the the intensity from the final scanning and the third will be a subtraction of one from the other. Here, have a look into the directory "170929-tutorial-microwell-1/results/". Any directory in the results directory should have a "intensities.tcv" and a "meta.tsv". the intensities file contain gal file like intensities. The meta file contains the processing type and a comment. The processing type names the method how the intensities were generated (e. g. Florescence Reader, Genepix, Image2Numeric0.1, ...). The comments can be used to described more in detail how the intensity values have been obtained. here, the meta example of the substration reslt. The other once can be find in here .
processing_type Fluorescence Reader
comment The values for intensities have been obtained by subtracting the intensities obtained by the fluorescence reader before and after the incubation.
