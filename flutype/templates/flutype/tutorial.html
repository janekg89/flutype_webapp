Glossary

In the following an overview about the main concepts is given and how they are used.
termexplanation
batch A batch is a solution used in a measurement or experiment. Mostly these are ligand batches i.e, solutions of a given ligand, e.g., peptides, viruses, or antibodies.
ligand A ligand is the main object used in a binding assay. Ligands can be peptides, viruses, or antibodies. To refer to the actual solution of a ligand use the batch concept.
measurement A measurement is a single experiment, i.e., a single microwell or peptide array which was measured. Multiple measurements can be combined to a study.
process A process describes the workflow in a measurement and consists of individual process steps. In a nutshell, the process describes all the individual steps taken in an experiment and in which order.
process step A process step describes a single step like washing, drying, incubation, ... in the workflow of doing a measurement.
study A study is a collection of multiple measurements, which were performed to answer a common question. E.g., multiple Elisa experiments to determine the concentration dependence of various viruses.

Tutorial: How to upload data

Uploading data into the database consists of two main parts, filling in generic data reused in multiple experiments, and experiment or measurement specific data:

  • The first part is the creation of generic data which is used in multiple measurements or studies.
    This generic data is reused, e.g., ligands like peptides or viruses are only defined once, same for the respective batches (i.e., solutions of peptides, viruses and antibodies), or the experimental steps (so called process steps) which are used in multiple measurements or studies.
    This generic data can be added/modified directly in the browser via the web interface (for instance by adding new peptides or viruses). A user must be logged in to add/modfiy this data.
  • The second part is the upload of data specific for a given measurement/experiment.
    For the upload the information must hereby be provided in a specific directory structure, i.e., certain files with defined names must exist in a given folder.
    When creating the files in this specific directory structure, it is important to check that the generic ligands (peptides, viruses, antibodies, ...), the used batches (solutions of the ligands) and the experimental steps (process steps) have already been uploaded in the database (see generic information above).

The largest unit in the database is a so called study which refers to a collection of measurements. This tutorial leads through the workflow of uploading experimental data for an example study containing measurements from an ELISA binding assay (microwell plate), a peptide-virus binding experiment (microwell plate), and a peptide-virus binding experiment on a peptide-array.

The specific directory structure for this example study is provided in the following supplementary file: flutype-tutorial.zip . The example study was uploaded in the FluTypeDB an is available here . The directory tree of the study can be seen here.

In the following a step by step tutorial to upload the example study is provided:

1. Study

The name of the folder in the directory will be displayed in the database as the sid (identifaiction string) of a study. I recommend to find a convention for the naming of the studies to not get name conflicts. Since our study is a tutorial, I name my study: "170929-tutorial". A study is build up off three blocks: metadata, measurements, and raw docs. Let's start with the metadata. Don't worry if you do not have results of a study while writing the metadata. They can be updated afterwards. Actually all of the meta fields are optional. Results related documents can be uploaded into the folder "raw_docs". The raw docs folder will be explained later. Now back to the metadata of our example. 2. Elisa Measurement

For the purpose of adding measurements to a study, one has to create the folder "measurements" in the main folder of the study and then add the folders, named accordingly to the measurements, into the measurements folder. For any measurement, the same five pieces of information are important: metadata, ligands, process, results, and raw_docs, results and raw_docs are optional. We start with an elisa measurement without any peptides involved. Let's begin with the most general measurement related information, the metadata.

2.1 Elisa Meta

batch_sid 12345689
measurement_type elisa
functionalization No
manufacturer Thermo F96 Maxisorp
The batch sid refers to the batch of the microwell plate. The measurement type has to be chosen from three choices: elisa, microarray and microwell. Both, microarray and microwell refer to fluoresence measurements. The functionalization refers to the surface substance of the microwell plate. In this measurement no ligands were bound to the surface of the microwell plate. Therefore, also no fuctionalization was necessary. We enter a "No", which is different to an empty space. An empty space refers to an unknown value ( NaN ). The manufacturer refers to the manufacturer of the microwell plate. These are the metadata of a measurement. This table is saved as "meta.tsv" in the main directory of a measurement folder. Subsequently, we continue with the next important measurement related information: the ligands and their positions on the microwell plate.

2.2 Elisa Ligands

Let's start with some general information related to ligands. Peptides, antibodies, viruses and any combination of several ligands (complexes) are referred to as ligands. Each of them is an ligand type and includes its specific information. A peptide, for example, is described by a linker, spacer, sequence and c-terminus. Besides the specific information, all ligands share the common fields: sid and comment. However, it might be interesting also to conduct measurements with e.g. different concentrations of the ligands, which is not captured by the ligands related pieces of information. Here, the concept of ligand batches comes into play. A ligand batch links a ligand to the following batch related properties: labeling, concentration, buffer, pH, purity and production date. Also for ligand batches, a sid has to be defined. Keep in mind that ligand batch sids are particularly important, because they are later on referenced in the gal files. Ligand batches, can be of five different types depending on what kind of ligand they are linked to: virus batch, peptide batch, antibody batch, complex batch and buffer batch. The buffer batch is a batch without a link to a ligand. The remaining four should be self explaining.

Now back to our measurement. In this example we do not immobilize any ligands on the surface. This reduces our work. No, need to define any peptides or peptide batches. However, still we are supposed to deliver a so called gal file, which maps a ligand batch to a location on a microwell plate. Since no ligands are present in any well, we fill all entries in the table with "No". Note that the table has to have the same dimensions as the microwell plate. For microarrays it might be more reasonable to provide a gal file, which is fine too. This table is saved as "lig_fix.txt" in the main directory of the measurement.

2.3 Elisa Process

Before digging into the specific process of our elisa measurement, let's start with a general description of a process. The process is described by process steps within a measurement. A process step can be of various types: blocking, drying, incubating-analyt, incubating, quenching, scanning, spotting and washing. Let me know if you have any suggestions for more. Each step is described by a sid , method, temperature and comment. Some of the steps take also the information substance and duration. Let's take a look at each step.

A process can contain many steps in an arbitrary ordering and with additional measurement specific pieces of information. We explain this concept based on our example. 2.4. Elisa Results

In this section, we are going to deal with pieces of information related to the measurement results. Measurement results describe the intensities resulting from the measurement. In this example, we measured the intensities during the last step of our process and saved tem as "intensity_final.tsv" in the main folder of the measurement. If we want to use these intensities as results we have to save it again in a folder within the results folder. It might seam to be a redundant procedure. However, the aim is to decouple the results from the process. In the microwell example, this approach might become clearer. There, multiple scans have been performed during one process. From theses scans different results have been obtained. Often, it might be interesting to subtract the background or to do even more sophisticated post-processing on the measured intensities and to show the output in as results. This concept makes it possible to do just that independently from the measured intensities. Again, the "sid" of the measurements results are defined by the folder name. This time the folder has to placed into the folder "results". We decided to call the result "raw". A results folder contains,besides the intensities, also some metadata related to the result. The following table shows the "meta.tsv" saved in our results folder.

3.4 Microwell Results

In this example, we want to show how to upload three results. The first result will be the intensities from the the first scanning (process step 7), the second result will be the the intensity from the final scanning and the third will be a subtraction of one from the other. Have a look into the directory "170929-tutorial-microwell-1/results/" in the directory tree . Any directory in the results directory should have a "intensities.tsv" and a "meta.tsv". the intensities file contain gal file like intensities. The meta file contains a , "comment", "processing type" and "intensity_file" name. The processing type names the method how the intensities were generated (e. g. Florescence Reader, Genepix, Image2Numeric0.1, ...). The comments can be used to described more in detail how the intensity values have been obtained.
